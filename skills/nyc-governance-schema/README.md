# NYC Governance Schema Skill

Project-specific skill for the NYC Governance Organizations dataset schema validation and Phase II field implementation.

## Purpose

This skill provides specialized guidance for working with the NYC Governance Organizations schema, ensuring data quality and consistency during Phase II expansion.

## Key Features

- **Phase II Schema Implementation**: Guidance for new fields (reports_to redefinition, org_chart_oversight, authorizing_authority, authorizing_url, appointments_summary)
- **Data Validation Rules**: Comprehensive validation standards for all schema fields
- **Entity Classification**: Standardized categorization of NYC governance entities
- **Relationship Mapping**: Clear distinction between legal reporting and administrative oversight
- **Research Protocols**: Source verification and quality assurance guidelines

## Phase II Focus Areas

### Schema Expansion
- 5 new/redefined fields to better capture organizational relationships
- Clear separation of legal vs administrative reporting structures
- Enhanced appointment authority tracking

### Data Quality
- Mandatory field population standards
- Cross-reference validation requirements
- Source documentation standards

### Mayor's Office of Appointments Integration
- Comprehensive appointment mechanism documentation
- Complete coverage validation for MOA scope
- Authority relationship mapping

## Usage

This skill should be active during all Phase II data collection, validation, and schema implementation work to ensure consistency and accuracy across the expanded NYC Governance Organizations dataset.

## Dependencies

- Integrates with civic-ai-tools legal document formatting skill
- Works alongside MOA research protocol skill
- Supports NYC legal citation skill standards
